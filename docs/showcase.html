Three layers of trust

Credentials isolated. Boundaries enforced. Everything visible.

Credential isolation

Your password stays in an isolated cloud browser. The agent gets a pre-authenticated session — works inside your account without knowing how to get in.

Cloudflare-trusted, SOC2 Type II certified. Session ends → access revoked.

Scoped boundaries

Safe by default. Read inbox? Yes. Delete contacts? Blocked. No LLM in the loop — rules checked in microseconds.

Pattern matching. No hallucination, no latency.

Full visibility

Every action logged — allowed and blocked. Budget enforced. Instant revocation.

One command to see. One command to stop.

Five minutes to safe access

No passwords shared. No Chrome profiles. No blind trust.

1

Set up an identity

Log into your accounts once in an isolated cloud browser. Your password stays there — the agent never sees it.

2

Define boundaries

Write a shield.yaml — what the agent can do, what it can't, how long it lasts. Anything not allowed is blocked.

3

Let it work

Scoped session. Already logged in. Limited to your rules. Every action logged. One command to stop.

Why this exists — and why now

declawed is Behalf's trust infrastructure, open-sourced. A proof of concept for the thesis that delegation needs a control layer.

The tools exist.
The controls don't.

OpenClaw: 200K GitHub stars. Creator acqui-hired by OpenAI. People adopt agents even when they break things — because delegated work is too compelling to resist.

But nobody wants agents running wild on Salesforce, Gmail, or their CRM. They want scoped sessions, visibility, and a kill switch.

declawed is proof it's solvable — open source because the problem is too urgent for a waitlist.
